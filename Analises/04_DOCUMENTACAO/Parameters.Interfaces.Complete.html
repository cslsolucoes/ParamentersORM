Interface IParametersDatabase

A interface IParametersDatabase fornece acesso completo a parâmetros armazenados em banco de dados. Suporta múltiplos engines (UNIDAC, FireDAC, Zeos) e múltiplos tipos de banco (PostgreSQL, MySQL, SQL Server, SQLite, FireBird, Access, ODBC).

Características:
  • Suporte a múltiplos engines de banco de dados
  • Suporte a múltiplos tipos de banco
  • Criação automática de tabela (opcional)
  • Reordenação automática de parâmetros
  • Thread-safe com TCriticalSection
  • Fluent Interface para código legível

Configuração

function TableName(const AValue: string): IParametersDatabase; overload fluent
function TableName: string; overload
Finalidade: Define ou obtém o nome da tabela onde os parâmetros são armazenados.

Parâmetros:
Parâmetro Tipo Descrição

Retorno:
  • Setter: Retorna Self para encadeamento
  • Getter: Retorna o nome da tabela atual

Valor Padrão: 'config'

Exemplo:
// Define nome da tabela DB.TableName('config'); // Obtém nome da tabela var Table: string; Table := DB.TableName; // Retorna 'config'

Operações CRUD

List

function List: TParameterList; overload
function List(out AList: TParameterList): IParametersDatabase; overload
Finalidade: Lista todos os parâmetros ativos da tabela, respeitando os filtros de ContratoID, ProdutoID e Título.

Retorno: TParameterList com todos os parâmetros encontrados.

Importante: A lista retornada deve ser liberada manualmente com AList.Free ou AList.ClearAll.

Exemplo:
var ParamList: TParameterList; ParamList := DB.List; try for var I := 0 to ParamList.Count - 1 do ShowMessage(ParamList[I].Name + ' = ' + ParamList[I].Value); finally ParamList.Free; end;

Get

function Get(const AName: string): TParameter; overload
function Get(const AName: string; out AParameter: TParameter): IParametersDatabase; overload
Finalidade: Obtém um parâmetro específico pelo nome (chave).

Parâmetros:
Parâmetro Tipo Descrição
AName string Nome (chave) do parâmetro a ser buscado
AParameter out TParameter Parâmetro encontrado (retornado via parâmetro out)

Retorno: Instância de TParameter com os dados encontrados. Se não encontrado, retorna parâmetro vazio (não nil).

Nota: O método busca TODOS os registros (ativos e inativos) para permitir reativação de parâmetros deletados.

Exemplo:
var Param: TParameter; Param := DB.Get('database_host'); try if Param.Name <> '' then ShowMessage('Valor: ' + Param.Value) else ShowMessage('Parâmetro não encontrado'); finally Param.Free; end;

Insert

function Insert(const AParameter: TParameter): IParametersDatabase; overload fluent
function Insert(const AParameter: TParameter; out ASuccess: Boolean): IParametersDatabase; overload fluent
Finalidade: Insere um novo parâmetro na tabela. Se o parâmetro já existir (mesma chave + título + ContratoID + ProdutoID), a inserção falha silenciosamente.

Parâmetros:
Parâmetro Tipo Descrição
AParameter TParameter Parâmetro a ser inserido (deve ter Name, Value, Titulo preenchidos)
ASuccess out Boolean True se inserido com sucesso, False se já existir ou falhar

Comportamento:
  • Verifica se parâmetro já existe antes de inserir
  • Gerencia ordem automaticamente conforme configurações
  • Se ordem <= 0 e renumerar zero estiver habilitado, calcula automaticamente
  • Se ordem especificada e reordenação habilitada, ajusta ordens existentes
  • Garante que a tabela existe antes de executar INSERT

Exemplo:
var Param: TParameter; Param := TParameter.Create; try Param.Name := 'erp_host'; Param.Value := 'localhost'; Param.Titulo := 'ERP'; Param.Description := 'Host do servidor ERP'; Param.ContratoID := 1; Param.ProdutoID := 1; Param.Ordem := 1; Param.Ativo := True; var Success: Boolean; DB.Insert(Param, Success); if Success then ShowMessage('Inserido com sucesso!') else ShowMessage('Parâmetro já existe ou falha na inserção'); finally Param.Free; end;

Update

function Update(const AParameter: TParameter): IParametersDatabase; overload fluent
function Update(const AParameter: TParameter; out ASuccess: Boolean): IParametersDatabase; overload fluent
Finalidade: Atualiza um parâmetro existente na tabela. O parâmetro é identificado pelo campo Name (chave).

Comportamento:
  • Se o parâmetro não existir, a atualização falha (ASuccess = False)
  • Gerencia ordem automaticamente quando o título muda
  • Se ordem especificada e reordenação habilitada, ajusta ordens existentes
  • Atualiza campo data_alteracao automaticamente

Exemplo:
var Param: TParameter; Param := DB.Get('erp_host'); try if Param.Name <> '' then begin Param.Value := '192.168.1.100'; // Novo valor var Success: Boolean; DB.Update(Param, Success); if Success then ShowMessage('Atualizado com sucesso!'); end; finally Param.Free; end;

Delete

function Delete(const AName: string): IParametersDatabase; overload fluent
function Delete(const AName: string; out ASuccess: Boolean): IParametersDatabase; overload fluent
Finalidade: Remove um parâmetro da tabela (soft delete - marca como inativo).

Nota: O método faz soft delete, ou seja, marca o campo ativo como False. O registro permanece na tabela e pode ser reativado posteriormente.

Exemplo:
var Success: Boolean; DB.Delete('erp_host', Success); if Success then ShowMessage('Parâmetro removido (marcado como inativo)');

Interface IParametersInifiles

A interface IParametersInifiles fornece acesso a parâmetros armazenados em arquivos INI. Preserva comentários e formatação do arquivo original.

Características:
  • Preserva comentários e formatação do arquivo
  • Suporte a múltiplas seções (títulos)
  • Criação automática de arquivo (opcional)
  • Remoção automática de seções vazias
  • Thread-safe com TCriticalSection
  • Fluent Interface para código legível

Configuração

FilePath

function FilePath(const AValue: string): IParametersInifiles; overload fluent
function FilePath: string; overload
Finalidade: Define ou obtém o caminho completo do arquivo INI.

Exemplo:
Ini.FilePath('C:\Config\params.ini');

Section

function Section(const AValue: string): IParametersInifiles; overload fluent
function Section: string; overload
Finalidade: Define ou obtém a seção (título) padrão para operações. Se vazio, opera em todas as seções.

Exemplo:
Ini.Section('ERP'); // Opera apenas na seção [ERP]

AutoCreateFile

function AutoCreateFile(const AValue: Boolean): IParametersInifiles; overload fluent
function AutoCreateFile: Boolean; overload
Finalidade: Define se o arquivo deve ser criado automaticamente se não existir.

Valor Padrão: True

Exemplo:
Ini.AutoCreateFile(True); // Cria arquivo automaticamente

Operações CRUD

List / Get / Insert / Update / Delete / Exists

Finalidade: Operações CRUD similares à interface Database, mas operando em arquivos INI.

Diferenças Importantes:
  • Preserva comentários e formatação do arquivo
  • Remove seções vazias automaticamente ao deletar última chave
  • Permite chaves com mesmo nome em seções diferentes
  • Valida duplicatas apenas dentro da mesma seção

Exemplo:
// Lista parâmetros var ParamList: TParameterList; ParamList := Ini.List; try // Usa ParamList... finally ParamList.Free; end; // Obtém parâmetro var Param: TParameter; Param := Ini.Get('host'); try if Param.Name <> '' then ShowMessage(Param.Value); finally Param.Free; end;

Utilitários

function FileExists: Boolean; overload
function FileExists(out AExists: Boolean): IParametersInifiles; overload

Exemplo:
if not Ini.FileExists then ShowMessage('Arquivo não existe');

function Refresh: IParametersInifiles; fluent
Finalidade: Recarrega o arquivo INI do disco.

Exemplo:

Interface IParametersJsonObject

A interface IParametersJsonObject fornece acesso a parâmetros armazenados em objetos JSON. Suporta formatação JSON com indentação e múltiplos objetos (títulos).

Características:
  • Formatação JSON com indentação
  • Suporte a múltiplos objetos (títulos)
  • Criação automática de arquivo (opcional)
  • Remoção automática de objetos vazios
  • Suporte a encoding UTF-8 e ANSI
  • Thread-safe com TCriticalSection
  • Fluent Interface para código legível

Configuração

JsonObject / ObjectName / FilePath / AutoCreateFile

Finalidade: Configura o objeto JSON, nome do objeto (título), caminho do arquivo e criação automática de arquivo.

Exemplo:
// Configura objeto JSON Json.JsonObject(MyJsonObject) .ObjectName('Parameters') .FilePath('C:\Config\params.json') .AutoCreateFile(True);

Operações CRUD

List / Get / Insert / Update / Delete / Exists

Finalidade: Operações CRUD similares às outras interfaces, mas operando em objetos JSON.

Diferenças Importantes:
  • Formata JSON com indentação automática
  • Remove objetos vazios automaticamente ao deletar última chave
  • Permite chaves com mesmo nome em objetos diferentes
  • Valida duplicatas apenas dentro do mesmo objeto

Exemplo:
// Lista parâmetros var ParamList: TParameterList; ParamList := Json.List; try // Usa ParamList... finally ParamList.Free; end;
